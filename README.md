来源表：`PETBASE_CONF`

| `unit_type` | 属性 ID |
| `unit_type_name` | 属性名称 |
| `wish_number` | 噩梦进度条权重值 |

> 只含有 `wish_number` 的 747 只精灵，无此字段的精灵不参与噩梦机制。

### 5. `nightmare_trigger_conf.csv` — 噩梦触发配置

来源表：`NPC_REFRESH_BONUS_CONF`

噩梦触发的核心参数，只有 1 条记录。

| 字段 | 说明（推测） |
|---|---|
| `id` | 配置 ID |
| `editor_name` | 配置名称 |
| `event_type` | 事件类型 |
| `event_count_type` | 计数类型 |
| `bonus_type` | 奖励类型 |
| `bonus_id` | 奖励 ID |
| `is_repeat_bonus` | 是否可重复触发 |
| `probability` | 硬保底配置（event_count_value=100 时必触发） |
| `prob_type` | 概率类型（2=公式型） |
| `prob_param1` | 公式参数 a（[520]） |
| `prob_param2` | 公式参数 b（[46]） |
| `trig_time_reset_type` | 重置类型（7=周期重置） |
| `trig_time_reset_begin` | 重置起始时间 |
| `trig_time_reset_gap` | 重置间隔（7 00:00:00 = 每 7 天） |
| `bonus_event_petbase_field_count` | ~~热机次数（前 30 次捕获不判定噩梦）~~ **[勘误]** 滑动窗口大小。值为 30，表示计算本次捕获触发噩梦时，取最近 30 次捕获记录作为判定窗口（用于匹配球池的 `petbase_need_persent` 门槛），而非"前 30 次不判定"。配置表原始字段 `editor_name` 为"连续捕捉—参与事件计数遍历的最近数量" |
| `is_refresh_petbase_field_count` | 切换进化链时是否重置计数器 |

> **[勘误] 关于噩梦触发公式：**
>
> 视频中推测噩梦触发概率与 `wish_number` 相关，这很可能是错误的。根据社区大量实测数据，噩梦触发概率更可能与 CGR（基础捕捉率，即 `catch_guarant_rate`）相关，而 CGR 与 `wish_number` 恰好负相关（难抓的精灵 CGR 低、wish_number 高）。
>
> 支撑依据：
> - 75 次有记录的噩梦触发中，63 次出了恶魔狼，12 次歪了（16.0%）
> - 平均每次噩梦触发消耗 30.3 个高级球，中位数 30 球，标准差 14.5 球，最少 2 球，最多 72 球
> - 群友普遍反映高级球消耗 30 个左右出一个噩梦
> - 使用补光球（100% 捕捉率）时，6-10 个球出一个恶魔狼
>
> 这些数据表明：球的捕捉率越高，噩梦触发越快。补光球（100% 捕获）约 6-10 次触发，高级球约 30 次触发，与 CGR 正相关而非与 wish_number 相关。具体公式仍待进一步验证。

### 6. `monster_catch_conf.csv` — 精灵捕获率配置表

来源表：`MONSTER_CATCH_CONF`

每只精灵独立的捕获难度参数，共 2009 条记录。这是决定"一只精灵好不好抓"的核心数据。

| 字段 | 说明（推测） |
|---|---|
| `id` | 捕获配置 ID（不同于 petbase_id） |
| `name` | 精灵名称 |
| `Catch_Threshold` | 捕获阈值（越低越难抓） |
| `catch_guarant_rate` | 保底捕获率（万分比，180=1.8%） |
| `Catch_Ball_level` | 最低球等级要求 |
| `pet_settled_basic_reward` | 捕获结算基础奖励值（来自 `PETBASE_CONF`，按精灵名称匹配合并） |

**`Catch_Threshold` 档位分布：**

| Catch_Threshold | 精灵数量 | 难度 |
|---|---|---|
| 50000 | 20 | 噩梦精灵形态（极高阈值，特殊机制捕获） |
| 10000 | 2 | 特殊精灵 |
| 1000 | 471 | 普通精灵（最常见） |
| 500 | 276 | 较难捕获 |
| 250 | 553 | 稀有精灵（如恶魔狼） |
| 0 | 687 | 无阈值/不可捕获 |

**`pet_settled_basic_reward` 常见档位：**

| 奖励值 | 精灵数量（PETBASE） | 典型精灵 |
|---|---|---|
| 0 | 519 | 赤毛鸡仔等（无奖励/不可捕获） |
| 800 | 73 | 草头鸭、恶魔叮 |
| 1200 | 65 | 喵喵、圆眼蜘蛛 |
| 1400 | 59 | 遁鼠 |
| 2000~2100 | 117 | 蹦蹦草、遁地鼠 |
| 3000 | 76 | 水灵 |
| 4000 | 38 | 迪莫 |
| 6211 | 9 | 恶魔狼、裘卡老师 |

**怎么用这张表：**

想知道一只精灵好不好抓，主要看两个字段：
- `Catch_Threshold`：捕获阈值，越高越容易抓。喵喵是 1000，恶魔狼只有 250
- `catch_guarant_rate`：保底捕获率，多扔几个球后的兜底概率。喵喵是 1700（17%），恶魔狼只有 180（1.8%）

最终捕获概率由多个因素共同决定，不是简单的乘法关系：
- 球的参数（`ball_conf.csv` 的 `ball_prob`、`ball_threshold_modify`）
- 精灵当前血量（血越低越好抓，红区 ≤30% 时概率最高）
- 是否背击（从背后偷袭有概率加成）
- 连续失败累加（多次失败后有伪随机保底递增）
- 等级差（超过一定等级差后变难或无法捕获）

具体公式在服务端，客户端配置表无法看到完整算法。以上字段含义均为推测。

### 8. `球池加权概率表.csv` — 噩梦球池加权概率汇总（二次计算）

来源：基于 `bonus_pool_conf.csv` 和 `accu_pool_conf.csv` 的二次计算结果。

将 7230 个球的原始权重汇总为每个池子的加权概率，方便直接查阅"触发噩梦后，从某个池子里摸出异色的概率是多少"。共 7251 行（含表头），覆盖全部 38 个池子。

| 字段 | 说明 |
|---|---|
| `池子ID` | 池子标识（chaos / evo_xxx / skilldam_xxx） |
| `池子名称` | 池子中文名 |
| `池子总权重` | 该池所有球的权重之和 |
| `金球总权重` | 该池中 `is_rare=True`（异色）球的权重之和 |
| `金球总占比%` | 金球权重 / 池子总权重 × 100 |
| `此池单次闪率%` | 单次噩梦触发时从此池抽到异色的概率 |
| `含保底期望噩梦次数` | 考虑保底机制后，期望需要多少次噩梦触发才能出异色 |
| `精灵名` | 球对应的精灵名称 |
| `是否异色` | 是否为金球（异色球） |
| `权重` | 该球的权重 |
| `在总池中占比%` | 该球权重 / 池子总权重 × 100 |
| `在金球中占比%` | 该球权重 / 金球总权重 × 100（仅金球有值） |

> **注意**：这是唯一一份经过二次计算的衍生数据，不是原始配置表直接导出。计算逻辑见仓库中的脚本。

**怎么用这张表：**

想知道"我刷某条进化链 / 某个属性，噩梦摸出异色的概率多大"，直接查对应池子的 `此池单次闪率%` 和 `含保底期望噩梦次数` 即可。混沌池（chaos）是兜底池，无条件触发，单次闪率约 2.4%，含保底约 35 次噩梦出一只异色。

## 补充说明

- 除 `球池加权概率表.csv` 外，所有数据均从游戏客户端 JSON 配置表直接导出，未做任何二次计算
- **所有字段含义均为推测，不代表服务端实际逻辑，仅供研究参考**
